Connecting a PostgreSQL Cluster

Connect your PostgreSQL cluster to PGF WatchTower. Takes about 10 minutes, assuming the monitoring role is already created.

📅 Updated 3 June 2025
⏱ 12 min read
🔖 Installation

Prerequisites

Before connecting your cluster, verify the following:

  • PostgreSQL 12, 13, 14, 15, or 16 is running and accessible from the PGF Agent host
  • The PGF WatchTower agent is installed on the same host or a host with network access to port 5432
  • You have a PostgreSQL superuser account, or a dedicated monitoring role with the required grants (see below)
  • Outbound HTTPS (port 443) is open from the agent host to api.postgresfirst.com
ℹ️
We recommend creating a dedicated pgf_monitor role rather than connecting as a superuser. This limits blast radius in the event of a credentials compromise.

Creating the monitoring role

Run the following SQL on your PostgreSQL cluster to create the minimum-privilege monitoring role:

SQL
-- Create the dedicated monitoring role CREATE ROLE pgf_monitor WITH LOGIN PASSWORD 'your-secure-password'; -- Grant required permissions GRANT pg_monitor TO pgf_monitor; GRANT CONNECT ON DATABASE postgres TO pgf_monitor; -- Required for query statistics (pg_stat_statements) GRANT SELECT ON pg_stat_activity TO pgf_monitor; GRANT SELECT ON pg_stat_replication TO pgf_monitor; GRANT pg_read_all_stats TO pgf_monitor;

Connection Configuration

Open the PGF WatchTower agent configuration file. The default location is /etc/pgf/watchtower.yaml.

YAML
# /etc/pgf/watchtower.yaml cluster: name: "production-primary" host: "db.yourcompany.internal" port: 5432 database: "postgres" username: "pgf_monitor" password: "${PGF_MONITOR_PASSWORD}" # use env var, not plaintext ssl_mode: "require" collection: interval_seconds: 30 query_stats: true replication_stats: true vacuum_stats: true pgf_api: endpoint: "https://api.postgresfirst.com/v3" api_key: "${PGF_API_KEY}"

Configuration parameters

Parameter Type Required? Description
cluster.name string Required Display name for this cluster in the PGF dashboard. Must be unique per workspace.
cluster.host string Required Hostname or IP address of the primary PostgreSQL instance.
cluster.ssl_mode string Optional SSL mode for the connection. Accepts disable, require, verify-ca, or verify-full. Default: require.
collection.interval_seconds integer Optional Metric collection interval. Minimum 15, maximum 300. Default: 30.
collection.query_stats boolean Optional Enable collection of query statistics via pg_stat_statements. Requires the extension to be installed. Default: true.
⚠️
Do not put passwords in plaintext in the config file. Use environment variables (${VAR_NAME} syntax) or a secrets manager. The agent reads env vars at startup.

Verifying the Connection

After saving the configuration, restart the agent and check the connection status:

Shell
$ sudo systemctl restart pgf-watchtower $ pgf status # Expected output: ✓ production-primary connected PostgreSQL 15.4 last seen 2s ago ✓ api.postgresfirst.com reachable latency 42ms

If the connection is successful, the cluster will appear in your PGF dashboard within 60 seconds. The first metric collection runs immediately after connection.

✅
Once connected, navigate to WatchTower → Clusters in the dashboard. You should see your cluster listed with a green status dot and live metric graphs within 2 minutes.

Troubleshooting

If the agent fails to connect, run pgf diagnose for a detailed report of what passed and what failed. Common issues:

  • Authentication failure: Verify the pgf_monitor password and that the role has LOGIN privilege.
  • Connection timeout: Check that port 5432 is accessible from the agent host. Test with psql -h {host} -U pgf_monitor -p 5432 postgres.
  • SSL error: If your cluster requires client certificates, see Advanced SSL Configuration.
  • API key rejected: Confirm the key in your PGF workspace settings. Keys are workspace-scoped — don't mix production and staging keys.
